# F4 — List / Visualize Department Research Profiles (Team 5 — Backend)

## Overview

F4 exposes two read-only REST endpoints that give any visitor (no login required)
aggregated statistics about each academic department represented in SciHub.

**There is no `department` table in the database.** All data is derived at query
time from the existing `researcher_info`, `project`, `paper`, `author`,
`author_paper`, and `user` tables. The backend never writes to the database.

---

## New Database Tables / Columns

**None.** F4 is a read-only feature. No schema changes were made.

Key schema facts relied upon by the queries:

| Column | Table | Notes |
|--------|-------|-------|
| `department` | `researcher_info` | Stores the department name as a free-form string; acts as a natural group-by key |
| `school` | `researcher_info` | College/school the department belongs to |
| `research_fields` | `researcher_info` | Comma/semicolon-delimited keyword string |
| `is_active` | `project` | `varchar(255)`; accepted as active if `LOWER(is_active) = 'true'` or `= '1'` |
| `sponsor_organization_id` | `project` | `NULL` = unfunded; `NOT NULL` = funded |
| `year` | `paper` | `varchar(255)`; cast to `UNSIGNED` for numeric comparison |
| `first_name`, `last_name` | `author` / `user` | Matched case-insensitively + trimmed to link authors to users |

## New API Endpoints

### 1. List Departments

```
GET /department/departmentList
```

**Query parameters (all optional):**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `pageLimit` | Integer | all | Max items to return |
| `offset` | Integer | 0 | 0-based start index |
| `sortBy` | String | `name` | `name` \| `facultyCount` \| `activeProjectCount` \| `publicationCountLast3Years` \| `fundedProjectCount` |
| `order` | String | `asc` | `asc` \| `desc` |

**200 Response:**

```json
{
  "offset": 0,
  "items": [
    {
      "name": "Computer Science",
      "school": "Engineering",
      "facultyCount": 12,
      "activeProjectCount": 5,
      "publicationCountLast3Years": 23,
      "fundedProjectCount": 3,
      "topKeywords": ["machine learning", "deep learning", "algorithms", "nlp", "robotics"],
      "faculty": null,
      "projects": null
    }
  ],
  "count": 1,
  "limit": 1,
  "total": 8,
  "sort": "name asc"
}
```

**404 Response:** returned when `researcher_info` contains no department rows.

---

### 2. Department Detail

```
GET /department/departmentDetail/:departmentName
```

The `:departmentName` path segment is URL-decoded (UTF-8) before lookup, so
`Computer%20Science` resolves to `Computer Science`.

**200 Response:**

```json
{
  "name": "Computer Science",
  "school": "Engineering",
  "facultyCount": 12,
  "activeProjectCount": 5,
  "publicationCountLast3Years": 23,
  "fundedProjectCount": 3,
  "topKeywords": ["machine learning", "deep learning", "algorithms", "nlp", "robotics"],
  "faculty": [
    {
      "userId": 42,
      "firstName": "Jane",
      "lastName": "Doe",
      "homepage": "https://example.com/jdoe",
      "researchFields": "machine learning, computer vision"
    }
  ],
  "projects": [
    {
      "id": 7,
      "title": "AI for Climate",
      "isActive": "true",
      "startDate": "2023-01-15",
      "principalInvestigatorId": 42
    }
  ]
}
```

**400 Response:** returned when the decoded department name is blank.

**404 Response:**

```json
"Department not found: NoSuchDept"
```

## Key Classes / Functions Added

### `models/Department.java`

Plain DTO (no `@Entity`) carrying aggregated department statistics.

| Field | Type | Description |
|-------|------|-------------|
| `name` | `String` | Department name (natural key) |
| `school` | `String` | College/school |
| `facultyCount` | `int` | Rows in `researcher_info` for this department |
| `activeProjectCount` | `int` | Projects with `is_active` = true/1 |
| `publicationCountLast3Years` | `int` | Papers within last 3 calendar years |
| `fundedProjectCount` | `int` | Projects with `sponsor_organization_id IS NOT NULL` |
| `topKeywords` | `List<String>` | Top-5 by frequency from `research_fields` |
| `faculty` | `List<DepartmentFaculty>` | Populated on detail only |
| `projects` | `List<DepartmentProject>` | Populated on detail only |

Inner class `DepartmentFaculty`: `userId`, `firstName`, `lastName`, `homepage`, `researchFields`.

Inner class `DepartmentProject`: `id`, `title`, `isActive`, `startDate`, `principalInvestigatorId`.

---

### `services/DepartmentService.java`

| Method | Purpose |
|--------|---------|
| `buildAllDepartments()` | Runs 5 aggregation queries via `Ebean.createSqlQuery`, merges results in a `LinkedHashMap` by lowercase+trimmed department name |
| `buildDepartmentDetail(String name)` | Calls `buildAllDepartments()`, matches by name, then populates `faculty` (via `ResearcherInfo.find`) and `projects` (via raw SQL) |
| `paginateAndSort(List, offset, pageLimit, sortBy, order)` | Sorts list in-place by one of 5 fields, paginates using `Common.paginate`, returns `RESTResponse` |
| `departmentList2JsonArray(List)` | Serialises department list to Jackson `ArrayNode` |
| `extractTopKeywords(String, int)` | Tokenises on `[,;]+`, lowercases, drops tokens shorter than 3 chars, returns top-N by frequency |

The five aggregation queries in `buildAllDepartments()`:

1. **Base** — `GROUP BY department, school` on `researcher_info` for faculty count
2. **Active projects** — JOIN to `project` filtering `LOWER(is_active) = 'true' OR is_active = '1'`
3. **Funded projects** — same JOIN filtering `sponsor_organization_id IS NOT NULL`
4. **Publications** — JOIN chain `researcher_info → user → author → author_paper → paper` with `CAST(year AS UNSIGNED) >= (currentYear - 3)`
5. **Keywords** — `GROUP_CONCAT(research_fields SEPARATOR ',')` per department

### `controllers/DepartmentController.java`

| Method | Route | Notes |
|--------|-------|-------|
| `departmentList(pageLimit, offset, sortBy, order)` | `GET /department/departmentList` | Returns `RESTResponse` envelope; 404 if no departments; 500 on exception |
| `departmentDetail(departmentName)` | `GET /department/departmentDetail/:departmentName` | URL-decodes name; 400 if blank; 404 if not found |

## Known Limitations / Future Improvements

- **Author–user matching is by name string.** The join `LOWER(TRIM(author.first_name)) = LOWER(TRIM(user.first_name)) AND ... last_name` will produce false positives when two people share the same name, and miss matches when names are entered differently (e.g., "Bob" vs "Robert"). A proper author-to-user ID mapping table would fix this.

- **`research_fields` is free-form text.** Keyword extraction is purely frequency-based with no NLP normalisation (e.g., "ML" and "machine learning" are counted separately). Tokens shorter than 3 characters are dropped — this intentionally excludes "AI" and "ML" as a workaround for generic noise, but also drops any legitimate 1–2 character keywords.

- **No caching.** Every API call re-runs all five aggregation queries. For large datasets this could be slow. Adding an in-memory cache with a TTL (e.g., Caffeine) would reduce DB load significantly.

- **Fuzzy `is_active` matching.** The query accepts `'true'` and `'1'`; values like `'True'`, `'yes'`, or `'1.0'` in existing data would be missed. This matches the pattern used elsewhere in the codebase.

- **No authentication enforced.** Per the F4 spec these endpoints are public. If that changes, a `@Security.Authenticated` annotation on the controller would be the correct mechanism.

- **`GROUP_CONCAT` truncation.** MySQL's default `group_concat_max_len` is 1024 bytes. Departments with many faculty members having long `research_fields` strings may have keywords silently truncated. Setting `group_concat_max_len = 65536` in application config would mitigate this.
